# ADR-001: SOP 2.0 母库 (Mother-Base) 初始化与自演进基座

## Status
Accepted

## Context
项目需要建立一个标准化的、可复用的、具有资产贡献能力的实验室（Foundry）。原有的扩展模式（Global Extensions）难以在不同环境下保持行为一致性，且缺乏将子库经验反哺给母库的闭环机制。我们需要将核心能力（Superpowers）本地化，并定义一套能够自我迭代的初始化与对齐规范。

## Decision
1. **能力本地化 (Localization)**：将 `gemini-superpowers` 扩展源码全部移入母库 `.gemini/skills/superpowers/`，解除对全局安装环境的依赖。
2. **Foundry-Workshop 架构**：
   - **Foundry (母库)**：定义全局 Schema（Env, Logic, Decision）和元技能。
   - **Workshop (子库)**：通过 `link.json` 挂载母库，执行环境对齐（Environmental Fingerprinting）。
3. **Skill TDD 规约**：所有的 Skill 编写必须遵循 `writing-skills` 定义的 TDD 流程（RED-GREEN-REFACTOR），并包含具体的 `Red Flags` 以约束 AI 行为。
4. **自演进闭环 (Self-Evolution)**：
   - 资产收割必须执行“去业务化（Decoupling）”和“参数化（Parameterization）”。
   - 任何架构偏差或错误纠正必须触发 **RCA (Root Cause Analysis)** 并记录在 `decisions/` 中。
   - 决策达成后必须同步更新（Patch）相关的 Skill 文档。

## Consequences
- **好处**：实现了资产的原子化沉淀，AI 在不同项目中具备了“长期记忆”和一致的协作模式；提高了逻辑提纯的纯度，降低了子库接入母库的门槛。
- **约束**：收割逻辑前必须编写 Vitest 测试规格；任何不符合 Schema 的代码提取将触发 Red Flags 并停止任务。
